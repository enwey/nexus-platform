# 项目任务拆解（执行版）

更新日期：2026-04-03

## 一、平台目标

围绕一个可商用的小游戏平台，完成以下主链路：

1. 开发者上传游戏包
2. 运营审核并发布
3. Android 发现/游戏库加载真实数据
4. 运行页下载游戏包并成功启动
5. 支持版本更新、强更、回滚

## 二、系统分层

- `backend`：账号、审核、分发、运营配置、版本策略
- `dev-portal`：开发者上传与版本管理
- `ops-portal`：审核、运营位配置、运行时运营控制
- `android-client`：玩家端 UI + runtime + bridge
- `mock-sdk`：`wx` 兼容接口

## 三、优先级拆分

### P0（已完成）

- JWT + Refresh + Redis 会话管理
- 下载链路改为后端代理流（默认）
- 发现页与游戏库真实数据接入
- Runtime Ops Console（运营后台）
- `game_ops_profile` 表与接口
- 运行页下载失败主链路修复

### P1（进行中）

- 运营素材在 Android 运行页全量消费
- 发现页运营能力细化（推荐位/卡片素材）
- 后台和 Android 国际化收口

### P2（待做）

- iOS 对齐与回归
- 自动化回归测试
- 发布与回滚脚本化

## 四、当前验收基线

- `localhost:8080/5173/5174` 可正常访问
- Android 真机（局域网）可加载真实数据
- 下载接口 `GET /game/download/{appId}` 返回 `200`
- 审核通过的游戏可启动

# 项目问题盘点与后续开发安排

## 文档目的

本文档用于记录 `nexus-platform` 当前的问题、已完成收口内容、剩余风险和后续开发安排，作为后续迭代的执行依据。

## 当前整体判断

仓库已经从“多个原型并排摆放”推进到“有根级合同、基础工程入口和主链路骨架”的状态，但距离真正的稳定交付还有几步。

当前主线状态如下：

- 代码主线：`backend` 和 `dev-portal` 已从源码损坏状态恢复，能够继续迭代
- 业务主线：登录、上传、审核、下载这条链路已基本拉通到后端和运营端，Android 已接上真实游戏列表与运行入口
- 工程主线：Android 真实构建、iOS 合同对齐、全链路验证仍未完成

## 本轮已完成的关键工作

### 阶段一：编译与源码收口

已完成：

- 修复 `backend` 中的主要编译阻塞
- 修复 `dev-portal` 中被乱码和坏模板破坏的页面源码
- 清理部分后端和前端源码编码问题
- 将 `dev-portal` 恢复到可构建状态

结果：

- `dev-portal` 构建通过
- `backend` 主代码已从明显损坏状态收口为可继续开发状态

结果：

- `npm run check:backend-contract` 通过
- `npm run build:dev-portal` 通过

## 当前剩余问题

### P0：仍阻塞真实交付的问题

#### 1. Android 仍未完成真实构建验证

- 当前仍缺少 `gradle/wrapper/gradle-wrapper.jar`
- 没有完成 `assembleDebug` 级别的真实验证
- 现阶段 Android 只能认为“静态主链路已接上”，还不能认为“客户端可交付”

#### 2. backend 仍未完成真实 Maven 构建验证

- 当前环境里没有 `mvn`
- 代码已做静态收口，但还没有得到一次真实后端构建结果

#### 3. 端到端运行仍未完成实机或模拟器验证

- Android 已接上后端列表和运行入口
- 但 아직没有完成“后端启动 -> 上传游戏 -> 审核通过 -> Android 下载 -> 解压 -> 运行”的完整验证

### P1：架构与合同仍需继续收束的问题

#### 4. iOS bridge 合同仍未对齐

- 根级合同检查仍会被 iOS bridge 一侧拖住
- 当前主要问题不是 Android 和运营端，而是 iOS 桥接能力矩阵仍未完全收敛

#### 5. token 仍是内存会话

- 当前 token 可用于本地联调
- 但服务重启后会失效
- 尚未进入生产可用的 JWT 或持久化 session 模型

#### 6. Android UI 仍有大量骨架代码

- 当前已经能承接真实列表和运行入口
- 但仍存在不少占位页面和简化过的 Compose 结构
- 重点在“可跑通”，不是“完整产品 UI”

## 后续修改安排

### 阶段三：收束平台契约与多端桥接

目标：让 SDK、Android、iOS 的桥接能力在合同和运行时行为上保持一致。

任务：

- 修复 iOS bridge 与合同不一致的问题
- 明确同步桥接在各端的支持矩阵
- 补充合同字段，区分 `implemented`、`async-only`、`stub`、`unsupported`
- 恢复根级合同检查通过

完成标准：

- `npm run check:contracts` 恢复通过
- 不再出现“合同写已实现，但运行时仍缺失”的情况

### 阶段四：补齐工程交付能力

目标：让仓库从“可继续开发”进入“可交付工程”状态。

任务：

- 补齐 Android 官方 `gradle-wrapper.jar`
- 跑通 Android `assembleDebug`
- 完成 backend 真实构建验证
- 明确 iOS 可执行构建入口
- 整理更稳定的联调说明

完成标准：

- Android 可真实构建
- backend 可真实构建
- 新成员可按文档完成本地联调

### 阶段五：全链路联调与测试

目标：让“上传 -> 审核 -> 下载 -> 运行”具备真实验证能力。

任务：

- 启动 backend 与基础设施做一次真实联调
- 使用管理员账号完成上传与审核
- 在 Android 模拟器或真机上验证下载和运行
- 补 controller / service / portal 的基础测试
- 条件允许时增加 Android / iOS CI 校验

完成标准：

- 完成至少一次完整业务主链路验证
- 关键主链路不再只依赖静态分析

2. 再补 Android / backend 的真实构建能力
3. 再做完整联调验证
4. 最后补测试、CI 和更完整的产品功能

## 结论

项目当前已经跨过了“源码损坏、后台无法构建、登录链路是假闭环”的阶段，开始进入“主链路已接通，但真实交付与多端一致性仍待完成”的阶段。下一步最重要的不是继续堆功能，而是把构建验证、iOS 合同和 Android 真运行这一层补齐。

# Project Nexus: 独立小游戏平台全栈开发计划 (全原生+Java版)

## 1. 项目概述

### 1.1 项目愿景
构建一个脱离微信生态的独立小游戏平台，让开发者能将 Cocos/Laya 开发的小游戏以 H5 格式打包上传，并在原生 App 中通过注入 wx-mock-sdk.js 伪装环境，实现零成本迁移。

### 1.2 核心架构
- **加载机制**: App 下载 ZIP -> 本地解压 -> 通过原生拦截器以虚拟域名访问本地文件，解决 CORS 跨域
- **通信机制**: 自研 NexusBridge。JS 调用 wx.xxx -> SDK 拦截 -> 转发给 Native -> Native 处理或请求 Java 后端/Nakama -> 结果异步回调给 JS
- **后端中台**: Java Spring Boot 处理业务逻辑；Nakama 处理游戏社交逻辑（排行榜、好友、云存档）

### 1.3 技术栈
- **Mock SDK**: TypeScript + Vite (构建目标: 单个 IIFE 格式 JS)
- **Android 端**: Kotlin + 原生 WebView + WebViewAssetLoader
- **iOS 端**: Swift + WKWebView + WKURLSchemeHandler
- **后端**: Java 17+, Spring Boot 3, PostgreSQL, Redis, MinIO (对象存储)
- **游戏服务**: Nakama (开源游戏服务器) + Docker
- **开发者后台**: Vue 3 + Element Plus

### 1.4 目录结构
```
/nexus-platform
  ├── /mock-sdk           # TS 编写的 wx API 拦截层
  ├── /android-client     # Android Kotlin 原生工程
  ├── /ios-client         # iOS Swift 原生工程
  ├── /backend            # Java Spring Boot 业务后端
  ├── /dev-portal         # Vue 3 开发者中心前端
  └── /docker-compose     # Nakama, Postgres, MinIO 部署配置
```

## 3. 执行约束

### 3.1 技术约束
- **禁止使用本地 HTTP 服务器**: Android 必须使用 WebViewAssetLoader，iOS 必须使用 WKURLSchemeHandler
- **优雅降级**: wx-mock-sdk 对于未实现的 API 必须返回 null 并打印 warn，严禁抛出异常导致 JS 挂起
- **安全**: 游戏 ZIP 包下载后需进行 MD5 校验；Webview 禁止访问非白名单的外部 URL
- **Java 规范**: 使用 Spring Boot 3.x, 使用 Result<T> 统一响应格式

### 3.2 质量约束
- 代码必须通过 ESLint/Checkstyle 检查
- 单元测试覆盖率 > 80%
- 所有 API 必须有文档注释
- 关键功能必须有集成测试

### 3.3 性能约束
- 游戏加载时间 < 3秒
- API 响应时间 < 200ms
- 内存占用 < 200MB
- APK/IPA 包大小 < 50MB
